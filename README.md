04-1. 단계별 프롬프트 전략
단계별 프롬프트 전략이란?
PRD를 기반으로 프로젝트를 여러 단계로 나누어 AI에게 구체적인 목표를 제시하는 방식입니다. (교재 p.145)
기획과 구현의 분리:
기획 (클로드 웹)구현 (클로드 코드)PRD 작성뼈대 → 기능 → UI → 고급 → 최적화
왜 단계별로 나눠야 할까?

한 번에 모든 기능을 요청하면 품질이 떨어짐
각 단계의 결과를 확인한 뒤 다음 단계 진행 가능
이전 단계의 결과물 위에 점진적으로 기능 추가

할 일 관리 앱을 만들고 싶어. PRD를 작성해줘.
매일 10~20개 정도의 할 일을 관리하는 개인용 앱이야.
주요 기능은:
- 할 일 추가, 수정, 삭제
- 완료 체크 기능

기술적으로 복잡하지 않게 순수 자바스크립트로 만들고 싶어.

04-2. 세션 관리와 컨텍스트
세션(Session)이란? (교재 p.147)
클로드 코드에서 대화를 시작해 명령을 주고받는 동안 유지되는 상태를 말합니다.
왜 세션 관리가 중요한가?

터미널을 닫으면 이전 대화가 사라진 것처럼 보임
실제로는 모든 대화 내용이 프로젝트 폴더에 자동 저장
며칠 후 이어서 작업해야 할 때 세션 재개가 필수


장기 프로젝트를 진행할 때는 세션과 컨텍스트 관리에 특히 주의해야 합니다!


세션 재개하기 (교재 p.148~149)
방법 1: claude --resume
이전 대화 기록을 불러와서 마지막 상태 그대로 이어서 작업할 수 있습니다.
방법 2: /resume 명령어 (별칭: /continue)
claude 실행 후 /resume을 입력하면 이전 대화를 선택하여 재개합니다.
방법 3: --continue 또는 -c 옵션
claude --continue
가장 최근의 마지막 대화를 자동으로 복구합니다.

자동 승인 모드 (교재 p.150)
Shift + Tab 키를 누르면 자동 승인 모드로 전환됩니다.

파일 수정 승인을 매번 누르지 않아도 자동으로 처리
반복적인 승인 요청을 생략하여 작업 속도 향상
다시 Shift + Tab을 누르면 일반 모드로 복귀

확인 표시: >> accept edits on (shift+tab to cycle)

편리하지만 코드 변경 사항을 나중에 반드시 점검하세요!


컨텍스트란? (교재 p.153~154)

현재 대화와 작업 정보를 저장 및 표시한 메모리 공간
대화가 길어지면 컨텍스트가 부족해져 자동 압축(auto-compact) 경고 발생

/context — 컨텍스트 상태 확인
현재 컨텍스트 사용량을 컬러 그리드로 시각화합니다.

컨텍스트 압축하기 (교재 p.155~156)
/compact — 대화 기록 요약
대화 기록을 요약하여 컨텍스트 공간을 확보합니다.

압축 후에도 CLAUDE.md와 프로젝트 파일은 유지됩니다. 작업 단위가 끝날 때마다 /compact로 정리하는 습관을 들이세요!

핵심 키워드 정리
이론 (교재 p.145, 157)

단계별 프롬프트 전략 — PRD를 기반으로 프로젝트를 여러 단계로 나누어 구체적인 목표를 제시하는 프롬프트 설계 방식
PRD — Product Requirements Document, 제품의 목적·기능·기술 스펙을 체계적으로 정리한 문서
세션(Session) — 클로드 코드에서 대화를 시작해 명령을 주고받는 동안 유지되는 상태
컨텍스트(Context) — 현재 대화와 작업 정보를 저장 및 표시한 메모리 공간
멀티모달 AI — 텍스트뿐만 아니라 이미지, 음성 등 서로 다른 형태의 데이터를 함께 인식할 수 있는 AI

실습 키워드 ① (교재 p.157)

--resume — 이전 대화 목록에서 원하는 세션을 선택해 복원
--continue (-c) — 가장 최근 대화를 자동으로 복구
자동 승인 모드 — Shift + Tab으로 전환, 반복적인 권한 승인을 생략
/compact — 대화 기록을 요약해 컨텍스트 공간을 확보

실습 키워드 ② (교재 p.171)

@ 기호 — 파일 및 폴더를 즉시 참조하고 분석
/config — 프로젝트 환경 설정을 확인하고 관리
/doctor — 시스템 상태를 진단하여 버전, 설치 방식, 업데이트 등을 점검
/export — 작업 기록을 클립보드나 파일에 저장하고 문서화
